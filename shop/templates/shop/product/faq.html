Webviews FAQ

Frequently Asked Questions about Messenger Bot Webviews

Webviews

What are webviews?

Technically a webview is simply a view in Messenger that displays a webpage.

How to get a webview?

Simply enter a URL in one of your bots' buttons, set the webview height accordingly, and DON'T FORGET to add the domain of the webview to the Messenger Extension whitelist. In Chatfuel's case you can find it at the bottom of the Configure page as "Domain Whitelisting".

How to customize a webview?

Some webview tools allow you to alter the way the webview looks through the URL by passing different parameters, but in most cases you need to have access to the HTML code and modify the webview from there.

How to pass data between the bot and the webview?

The easiest way would be to use the chatbot builder platform's API to send data from the webview's backend to the bot. In Chatfuel's case this can be done with the Broadcasting API.

Do I need a server to have webviews in my bots?

Not necessarily, if you just want to display static info to your users or have them enter some basic data in the webview and return it to the bot you can find services online that provide these simple webviews. If you want more complex webviews that store persistent data in your database for example then yes you need a server and a developer as well.
